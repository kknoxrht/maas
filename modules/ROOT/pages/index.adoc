= Red Hat OpenShift AI: Solving the GPU Silo with MaaS Enablement
:navtitle: Home

[.lead]
*Transform GPU infrastructure from isolated team silos into a centralized AI utility grid with governed token-based consumption.*

== Course Description

Organizations face a "GPU Silo" crisis where expensive, limited hardware is trapped within individual teams, creating simultaneous idle capacity and project bottlenecks. This course equips Platform Engineers, AI/ML Platform Administrators, and Developers with the expertise to design, deploy, and manage a scalable Models-as-a-Service (MaaS) ecosystem within Red Hat OpenShift AI.

By the end of this training, you will transition your infrastructure from "Private Power Generators" (siloed hardware) to a centralized "Municipal Utility Grid" with robust governance, usage quotas, and seamless integration of production-ready AI endpoints.

**Duration**: 8-12 hours

== What You'll Learn

This course covers:

* **Architecture**: Understanding the MaaS stack (KServe, Kuadrant, Gateway API)
* **Platform Administration**: Configuring CRDs and troubleshooting component enablement
* **Model Deployment**: Publishing models to the governed catalog with distributed inference
* **Governance**: Creating tiers, enforcing rate limits, and tracking token consumption
* **API Integration**: Consuming OpenAI-compatible endpoints with proper error handling

== Prerequisites

This course assumes you have:

* Red Hat OpenShift administration experience (Version 4.20+)
* Basic Kubernetes operator management knowledge
* Familiarity with core GPU concepts and generic GPU enablement
* A deployed instance of Red Hat OpenShift AI 3.3 or later

== Business Value

This course addresses critical organizational challenges:

**Without MaaS**:
* GPU hardware sits idle while projects queue
* Multiple teams deploy the same models redundantly
* No audit trail for compliance requirements
* Cannot track usage or forecast compute needs

**With MaaS**:
* Centralized catalog eliminates redundant deployments
* Fair resource allocation prevents team conflicts
* Complete audit logs and PII filtering via API gateway
* Token-based consumption enables accurate chargeback
