= Solving the GPU Silo with Model as a Service (MaaS)
:navtitle: Home

[.lead]
*Transform GPU infrastructure from isolated team silos into a centralized Centralized Model Hub with governed token-based consumption.*

== Course Description

Organizations face a "GPU Silo" crisis where expensive, limited hardware is trapped within individual teams, creating simultaneous idle capacity and project bottlenecks. 

This course equips Platform Engineers, AI/ML Platform Administrators, and Developers with the expertise to design, deploy, and manage a scalable Models-as-a-Service (MaaS) ecosystem within Red Hat OpenShift AI.

By the end of this training, you will transition your infrastructure from "dedicated, single-team clusters" (siloed hardware) to a centralized "consumable model backplane" with robust governance, usage quotas, and seamless integration of production-ready AI endpoints.

**Duration**: 3-4 hours

== What You'll Learn

This course covers:

 **Architecture**::
 Understand the transition from hardware management to token delivery, map the architectural layers (KServe, Kuadrant, Gateway API), and articulate how MaaS prevents "Shadow AI".

**Platform Administration**::
Configure the OpenShift AI CRDs to enable the MaaS component, establish Gateway API routing, and activate the dashboard interface.

 **Model Deployment**::
 Publishing models to the governed catalog with distributed inference

 **Governance**::
 Define tiered service limits, configure group-based rate limiting, track token consumption metrics, and automate chargeback workflows.

 **API Integration**::
 Navigate the Gen AI studio, generate authentication tokens, implement programmatic retry logic for rate limits, and integrate models into applications via OpenAI-compatible APIs.

== Target Audience

**Platform Engineers**::
Configure MaaS through the OpenShift AI operator
Manage governance, user groups, and observability
Monitor cluster health, gateway routing, and resource usage

**Developers**::
Consume MaaS endpoints and integrate models into applications


**Target Level**::
Tech Level 3 (Internal, Partners, and Customers)

== Prerequisites

This course assumes you have:

* Red Hat OpenShift administration experience (Version 4.20+)
* Basic Kubernetes operator management knowledge
* Familiarity with core GPU concepts and generic GPU enablement
* A deployed instance of Red Hat OpenShift AI 3.3 or later

== Business Value

This course addresses critical organizational challenges:

**Without MaaS**:

 * GPU hardware sits idle while projects queue
 * Multiple teams deploy the same models redundantly
 * No audit trail for compliance requirements
 * Cannot track usage or forecast compute needs

**With MaaS**:

 * Centralized catalog eliminates redundant deployments
 * Fair resource allocation prevents team conflicts
 * Complete audit logs and PII filtering via API gateway
 * Token-based consumption enables accurate chargeback
