= 5.3: Implement retry logic for rate limiting

Production applications must handle rate limiting responses (429 status) gracefully using exponential backoff retry logic.

When users exceed their tier's rate limit or quota, Kuadrant returns `429 Rate Limit Exceeded` with a `Retry-After` header indicating when to retry.

=== Understand rate limit responses

Make multiple rapid requests to trigger rate limiting (Bronze tier: 100 req/min):

[source,role="execute"]
----
for i in {1..10}; do
  curl -s -w "\nHTTP Status: %{http_code}\n" -X POST "https://${MAAS_URL}/v1/chat/completions" \
    -H "Authorization: Bearer ${MAAS_TOKEN}" \
    -H "Content-Type: application/json" \
    -d '{
      "model": "meta-llama/Meta-Llama-3-8B-Instruct",
      "messages": [{"role": "user", "content": "Hi"}],
      "max_tokens": 10
    }' | head -5
  sleep 0.5
done
----

If you exceed rate limits, you'll see:

----
HTTP Status: 429

{
  "error": {
    "message": "Rate limit exceeded. Retry after 45 seconds.",
    "type": "rate_limit_exceeded",
    "code": 429
  }
}
----

The response includes:

* HTTP status `429 Rate Limit Exceeded`
* `Retry-After` header with seconds to wait
* Error message explaining the limit

=== Implement exponential backoff retry logic

Create a Python script with proper retry logic:

[source,role="execute"]
----
cat > maas_client_with_retry.py <<'EOF'
#!/usr/bin/env python3
import os
import time
import requests
from typing import Optional

MAAS_URL = f"https://{os.getenv('MAAS_URL')}"
MAAS_TOKEN = os.getenv('MAAS_TOKEN')

def chat_completion_with_retry(
    model: str,
    messages: list,
    max_retries: int = 3,
    initial_backoff: float = 1.0
) -> Optional[dict]:
    """
    Make chat completion request with exponential backoff retry logic.

    Args:
        model: Model identifier
        messages: List of chat messages
        max_retries: Maximum number of retry attempts
        initial_backoff: Initial backoff delay in seconds

    Returns:
        API response dict or None if all retries failed
    """
    headers = {
        "Authorization": f"Bearer {MAAS_TOKEN}",
        "Content-Type": "application/json"
    }

    payload = {
        "model": model,
        "messages": messages,
        "max_tokens": 200
    }

    backoff = initial_backoff

    for attempt in range(max_retries + 1):
        try:
            response = requests.post(
                f"{MAAS_URL}/v1/chat/completions",
                headers=headers,
                json=payload,
                timeout=30
            )

            # Success
            if response.status_code == 200:
                return response.json()

            # Rate limit exceeded
            elif response.status_code == 429:
                if attempt < max_retries:
                    # Get retry-after header if present, otherwise use exponential backoff
                    retry_after = int(response.headers.get('Retry-After', backoff))
                    print(f"Rate limit exceeded. Retrying after {retry_after} seconds (attempt {attempt + 1}/{max_retries})")
                    time.sleep(retry_after)
                    backoff *= 2  # Exponential backoff for next attempt
                else:
                    print(f"Rate limit exceeded. Max retries ({max_retries}) reached.")
                    return None

            # Unauthorized or forbidden
            elif response.status_code in [401, 403]:
                print(f"Authentication error: {response.status_code}")
                print(response.json())
                return None

            # Other errors
            else:
                print(f"API error: {response.status_code}")
                print(response.json())
                return None

        except requests.exceptions.Timeout:
            print(f"Request timeout (attempt {attempt + 1}/{max_retries})")
            if attempt < max_retries:
                time.sleep(backoff)
                backoff *= 2
        except requests.exceptions.RequestException as e:
            print(f"Request failed: {e}")
            return None

    return None

# Test the retry logic
if __name__ == "__main__":
    print("Testing MaaS API with retry logic...\n")

    result = chat_completion_with_retry(
        model="meta-llama/Meta-Llama-3-8B-Instruct",
        messages=[
            {"role": "system", "content": "You are a helpful assistant for ChocoTech Industries."},
            {"role": "user", "content": "What are the benefits of dark chocolate?"}
        ],
        max_retries=3,
        initial_backoff=1.0
    )

    if result:
        print("Success!")
        print(f"Response: {result['choices'][0]['message']['content']}")
        print(f"\nTokens used: {result['usage']['total_tokens']}")
    else:
        print("Failed to get response after retries.")
EOF

== Troubleshooting

**Issue**: API returns 401 Unauthorized

**Solution**:

. Verify Bearer token is valid and not expired in Gen AI Studio → API tokens
. Check token is correctly set in environment variable: `echo ${MAAS_TOKEN:0:20}`
. Ensure Authorization header format: `Authorization: Bearer <token>` (not `Bearer: <token>`)
. Verify token was copied correctly (no extra spaces or newlines)

**Issue**: API returns 403 Forbidden

**Solution**:

. Your tier does not have access to the requested model
. Check model's tier access in Gen AI Studio → Models tab
. Verify your user group membership: `oc get user {user} -o yaml | grep groups`
. Check which tier your group is assigned to
. Request access from Platform Engineer to upgrade tier or publish model to your tier

**Issue**: API returns 429 Rate Limit Exceeded immediately

**Solution**:

. You've exhausted your tier's quota (requests/min or tokens/min)
. Check usage in Gen AI Studio → Usage tab
. Implement exponential backoff retry logic (see Exercise 3)
. Request tier upgrade from Platform Engineer if usage is legitimate
. Check `Retry-After` header for when quota resets

**Issue**: OpenAI SDK raises "Connection refused" error

**Solution**:

. Verify MaaS gateway route exists: `oc get route maas-default-gateway -n redhat-ods-applications`
. Check MAAS_URL environment variable is correct: `echo $MAAS_URL`
. Verify network connectivity from client to OpenShift cluster
. Check for firewall or proxy blocking HTTPS traffic
. Test with curl first to isolate SDK vs connectivity issues

**Issue**: Model inference returns empty or truncated response

**Solution**:

. Check `max_tokens` parameter - may be too low for complete response
. Increase `max_tokens` to allow longer completions (e.g., 500 or 1000)
. Check `finish_reason` in response - if "length", tokens limit was hit
. Verify model is healthy: `oc get llminferenceservice -n chocotech-models`
. Check model pod logs for inference errors: `oc logs -l app=llama-3-8b -n chocotech-models`

**Issue**: Requests timing out after 30 seconds

**Solution**:

. Large models or long prompts may take time to generate responses
. Increase request timeout in client code (e.g., `timeout=60`)
. Check model pod CPU/GPU utilization - may be overloaded
. Consider using streaming responses for better user experience
. Check for model pod issues: `oc get pods -n chocotech-models`
