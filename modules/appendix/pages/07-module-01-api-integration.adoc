= 5.1: Generate API tokens in Gen AI Studio

Before making API calls, developers need to generate Bearer tokens for authentication. Gen AI Studio provides a self-service interface for token generation.

API tokens authenticate users and determine their tier-based access level and quotas.

=== Navigate to Gen AI Studio

. Open the OpenShift AI dashboard: {rhoai_dashboard_url}
. Log in with your credentials:
  * Username: {user}
  * Password: {password}
. Click **Gen AI Studio** in the left navigation menu

You should see the Gen AI Studio interface with three tabs:

* **Models** - Discover available models in the catalog
* **API tokens** - Generate Bearer tokens for authentication
* **Usage** - View token consumption and quota status

=== Explore the model catalog

Click on the **Models** tab to see the catalog of available models.

You should see the models you deployed in Module 3:

----
Model: meta-llama/Meta-Llama-3-8B-Instruct
Status: Available
Tier Access: Bronze, Silver, Gold, Platinum
Endpoint: /v1/chat/completions

Model: nvidia/nemotron-4-8b-instruct
Status: Available
Tier Access: Bronze, Silver
Endpoint: /v1/chat/completions
----

Notice that models show which tiers can access them. Since your user is in the chocotech-quality-control group (Platinum tier), you can access all models.

== Exercise 2: Consume models via OpenAI-compatible API

Now that you have a Bearer token, you'll make REST API calls to list models and generate chat completions.

MaaS exposes models through the OpenAI-compatible API, enabling existing applications built for ChatGPT or GPT-4 to switch to self-hosted models by changing the base URL and Bearer token.

=== Set up environment variables

Save your Bearer token as an environment variable:

[source,role="execute"]
----
export MAAS_TOKEN="<paste-your-token-here>"
----

Replace `<paste-your-token-here>` with the actual token you copied from Gen AI Studio.

Get the MaaS gateway URL:

[source,role="execute"]
----
export MAAS_URL=$(oc get route maas-default-gateway -n redhat-ods-applications -o jsonpath='{.spec.host}')
----

Verify the environment variables:

[source,role="execute"]
----
echo "MaaS Gateway: https://${MAAS_URL}"
echo "Token starts with: ${MAAS_TOKEN:0:20}..."
----

Expected output:

----
MaaS Gateway: https://maas-default-gateway.apps.{cluster_domain}
Token starts with: eyJhbGciOiJSUzI1Ni...
----

=== List available models

Query the `/v1/models` endpoint to discover available models:

[source,role="execute"]
----
curl -s -X GET "https://${MAAS_URL}/v1/models" \
  -H "Authorization: Bearer ${MAAS_TOKEN}" \
  -H "Content-Type: application/json" | jq
----

Expected output (formatted JSON):

----
{
  "object": "list",
  "data": [
    {
      "id": "meta-llama/Meta-Llama-3-8B-Instruct",
      "object": "model",
      "created": 1714345678,
      "owned_by": "chocotech-models"
    },
    {
      "id": "nvidia/nemotron-4-8b-instruct",
      "object": "model",
      "created": 1714345690,
      "owned_by": "chocotech-models"
    }
  ]
}
----

This lists all models your tier can access. The API format matches OpenAI's `/v1/models` endpoint exactly.

=== Generate a chat completion

Make an inference request to `/v1/chat/completions`:

[source,role="execute"]
----
curl -s -X POST "https://${MAAS_URL}/v1/chat/completions" \
  -H "Authorization: Bearer ${MAAS_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "meta-llama/Meta-Llama-3-8B-Instruct",
    "messages": [
      {
        "role": "system",
        "content": "You are a helpful assistant for ChocoTech Industries, a chocolate manufacturer."
      },
      {
        "role": "user",
        "content": "What quality control metrics should we track for chocolate production?"
      }
    ],
    "max_tokens": 200,
    "temperature": 0.7
  }' | jq
----

Expected output (formatted JSON):

----
{
  "id": "chatcmpl-abc123",
  "object": "chat.completion",
  "created": 1714345700,
  "model": "meta-llama/Meta-Llama-3-8B-Instruct",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "For chocolate production quality control, you should track:\n\n1. Cocoa butter percentage and consistency\n2. Temperature during tempering (ideal: 27-31°C)\n3. Particle size distribution (ideally <20 microns)\n4. Moisture content (<2% for milk chocolate)\n5. Visual defect detection (blooming, cracks, color inconsistencies)\n6. Taste panel scores\n7. Packaging seal integrity\n\nThese metrics ensure consistent product quality and help identify issues early in the production process."
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 45,
    "completion_tokens": 127,
    "total_tokens": 172
  }
}
----

The response includes:

* **content**: The model's generated response
* **usage**: Token consumption (prompt_tokens + completion_tokens = total_tokens)
* **finish_reason**: Why generation stopped (stop = natural completion, length = max_tokens reached)
