# RHOAI Course Generation Prompt

Generate a complete course structure from a design document.

## Input Analysis

You will receive:
1. A design document (markdown) with course structure
2. An output directory path
3. Optional flags (--dry-run, --include-examples, --with-diagrams)

Parse the design document to extract:
- Course title
- Course goal (clear statement of what learners will accomplish)
- Target audience (roles, their perspective)
- Problem statement (why this course exists, business context)
- Performance objectives (learning outcomes)
- Module/chapter structure with sections and learning objectives

## Course Structure Mapping

Map the design document to Antora structure:

```
Design Input                 → Antora Output
─────────────────────────────────────────────────────────
Course Title                 → Course ID (rhoai3-{keyword})
                            → modules/ROOT/pages/index.adoc

Module 1                     → modules/ch1-keyword1/
  Section 1.1               → pages/s1-descriptive-name.adoc
  Section 1.2               → pages/s2-descriptive-name.adoc
  Section 1.3               → pages/s3-descriptive-name.adoc

Module 2                     → modules/ch2-keyword2/
  ...
```

## Key Mapping Rules

1. **Course ID**: Extract keyword from course title
   - "Fine-Tuning Language Models" → "fine-tuning"
   - "GPU Optimization Strategies" → "gpu-optimization"
   - Creates: `rhoai3-{keyword}/`

2. **Chapter Naming**: Map to directory names
   - Module 1: Fine-Tuning Economics → `ch1-economics`
   - Module 2: Data Preparation → `ch2-data-preparation`
   - Pattern: `ch{N}-{keyword}/`

3. **Section Naming**: From learning objectives
   - "Understand cost models" → `s1-cost-models.adoc`
   - "Execute fine-tuning jobs" → `s2-execution.adoc`
   - Pattern: `s{N}-{keyword}.adoc`

4. **Section Type Inference**: From learning objective keywords
   - "Understand", "Explain", "Describe" → presentation
   - "Deploy", "Configure", "Execute" → lab
   - "Test", "Assess", "Evaluate" → quiz
   - If unclear: presentation (safest default)

## File Contents

### antora.yml

```yaml
name: rhoai3-{keyword}
title: {Course Title}
version: '1.0'
nav:
  - modules/ROOT/nav.adoc
  - modules/ch1-{keyword}/nav.adoc
  - modules/ch2-{keyword}/nav.adoc
  [... one per chapter]
```

### package.json

```json
{
  "name": "rhoai3-{keyword}",
  "version": "1.0.0",
  "description": "{Course Title}",
  "scripts": {
    "build": "antora antora-playbook.yml",
    "watch:adoc": "chokidar 'modules/**/*.adoc' -c 'npm run build'",
    "serve": "python3 -m http.server -d build/site 8080"
  },
  "devDependencies": {
    "@antora/cli": "^3.1.0",
    "@antora/site-generator-default": "^3.1.0",
    "chokidar-cli": "^3.0.0"
  }
}
```

### .gitignore

```
node_modules/
build/

.DS_Store
.idea/
*.swp
*~
```

## ROOT Module Generation

Generate ROOT module content using:

1. **Course title** → Document title
2. **Course goal** → Lead opener (make it declarative, not "in this course you'll...")
3. **Problem statement** → Business value paragraph
4. **Target audience** → Definition list
5. **Performance Objectives** → Numbered list
6. **Modules list** → What's covered

## Chapter Index Generation

For each chapter:

1. **Chapter title** → Document title
2. **Module goal** (from design) → Chapter goal
3. **Sections** → Topics covered list
4. **Connection** → What's Next explaining next chapter's value

## Section Stub Generation

For each section in the design:

1. **Title** → From design document section name
2. **Time estimate** → Guess from section type
   - Presentation: 15-20
   - Lab: 45-60
   - Quiz: 10-15
3. **Lead opener placeholder** → "[Lead opener - replace with your content]"
4. **Learning objective** → In comments for reference
5. **What's Next** → Placeholder for next section

## Navigation File Generation

For each chapter, create nav.adoc mapping sections:

```
- Root level: Chapter title (index.adoc)
- Second level: Each section (s{N}-name.adoc)
- Titles must match section titles in design
- Cross-references must be correct paths
```

## Optional: Include Example Sections

If --include-examples flag:

1. Copy rhoai3-maas example presentation to s1-example-presentation.adoc
2. Copy rhoai3-maas example lab to s2-example-lab.adoc
3. Copy rhoai3-maas example quiz to s3-example-quiz.adoc
4. Add note: "These sections show RHOAI style and can be deleted or used as templates"
5. Renumber user sections to start from s4

Preserve example files so users can reference them while writing content.

## Optional: Create Diagram Placeholders

If --with-diagrams flag:

1. Create `images/ch{N}-diagrams/` for each chapter
2. For each major learning objective, create SVG stub:
   - "Understanding architecture" → `architecture-overview.svg`
   - "Cost calculation" → `cost-model.svg`
   - "Deployment flow" → `deployment-workflow.svg`
3. Include TODO comment in each SVG:
   ```xml
   <!-- TODO: Create diagram for [learning objective] -->
   <!-- Recommended: Excalidraw, draw.io, Lucidchart -->
   <!-- Save as SVG for version control -->
   ```
4. Create images/index.md with guidance on diagram tools

## Git Initialization

1. Initialize: `git init`
2. Create initial commit:
   ```
   git add .
   git commit -m "Initial course scaffold: {Course Title}

   - Generated from design document
   - Ready for content generation
   - Use create-rhoai-module to fill in sections"
   ```
3. Result: Course is version-controlled from start
