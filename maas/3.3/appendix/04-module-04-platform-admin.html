Exercise 4: Access the OpenShift AI dashboard

Now that MaaS is enabled and the dashboard is configured, access the web console to verify the Gen AI Studio interface is available.

The dashboard provides the self-service interface for discovering models, generating API tokens, and viewing usage metrics.

Get the dashboard URL

Find the OpenShift AI dashboard route:

oc get route rhods-dashboard -n redhat-ods-applications -o jsonpath='{.spec.host}'

Expected output:

rhods-dashboard-redhat-ods-applications.apps.{cluster_domain}

Or use the attribute placeholder directly: {rhoai_dashboard_url}

Log into the dashboard

  1. Open a web browser (or use the Showroom "OpenShift AI" tab)

  2. Navigate to the dashboard URL

  3. Log in with your OpenShift credentials:

    • Username: {user}

    • Password: {password}

You should see the OpenShift AI dashboard home page.

Navigate to Gen AI Studio

After logging in:

  1. Look for "Gen AI Studio" in the left navigation menu

  2. Click on "Gen AI Studio"

  3. You should see the Gen AI Studio interface with:

    • "Models" tab - Model catalog (currently empty until Module 3)

    • "API tokens" tab - Token generation interface

    • "Usage" tab - Consumption metrics (populated after API calls)

Figure 1. Gen AI Studio Interface in OpenShift AI Dashboard

Explore the interface

Take a moment to explore the Gen AI Studio interface:

  • Models tab: Will display the model catalog once models are deployed (Module 3)

  • API tokens tab: Where users generate Bearer tokens for API authentication

  • Usage tab: Shows token consumption, request counts, and cost metrics

Note: The Models tab will be empty until you deploy models in Module 3.

Verification

Confirm dashboard access:

  • Successfully logged into OpenShift AI dashboard

  • Gen AI Studio menu item is visible in left navigation

  • Gen AI Studio interface loads without errors

  • Models, API tokens, and Usage tabs are present

  • No error messages about missing components

Troubleshooting

Issue: MaaS API pod stuck in ContainerCreating

Solution:

  1. Check pod events: oc describe pod -l app=maas-api -n redhat-ods-applications

  2. Common causes:

    • Image pull issues - verify network connectivity to quay.io

    • Resource constraints - check node capacity

    • PVC binding issues - verify storage class availability

  3. Check operator logs for reconciliation errors: oc logs -n redhat-ods-operator deployment/rhods-operator

Issue: Gateway shows PROGRAMMED=False

Solution:

  1. Describe Gateway for error messages: oc describe gateway maas-default-gateway -n redhat-ods-applications

  2. Verify Service Mesh control plane is healthy: oc get smcp -n istio-system

  3. Check Istio operator logs: oc logs -n openshift-operators deployment/istio-operator

  4. Common cause: Service Mesh not installed or control plane not ready

Issue: Gen AI Studio menu item not visible after enabling

Solution:

  1. Verify OdhDashboardConfig was updated: oc get odhdashboardconfig odh-dashboard-config -o jsonpath='{.spec.dashboardConfig.genAiStudio}' should return true

  2. Force dashboard pod restart: oc delete pod -l app=rhods-dashboard -n redhat-ods-applications

  3. Clear browser cache and refresh dashboard

  4. Check dashboard pod logs for configuration errors: oc logs -l app=rhods-dashboard -n redhat-ods-applications

Issue: Dashboard returns 503 Service Unavailable

Solution:

  1. Verify dashboard pod is Running: oc get pods -l app=rhods-dashboard -n redhat-ods-applications

  2. Check dashboard service exists: oc get svc rhods-dashboard -n redhat-ods-applications

  3. Verify route is configured: oc get route rhods-dashboard -n redhat-ods-applications

  4. Check for NetworkPolicy blocking traffic

Issue: Kuadrant pods not deploying after MaaS enablement

Solution:

  1. Check DataScienceCluster status for errors: oc get datasciencecluster default-dsc -o yaml | grep -A 10 conditions

  2. Verify Kuadrant operator is installed: oc get csv -n openshift-operators | grep kuadrant

  3. If Kuadrant operator is missing, it should be installed automatically by OpenShift AI operator

  4. Check OpenShift AI operator logs: oc logs -n redhat-ods-operator deployment/rhods-operator | grep -i kuadrant

Issue: HTTPRoute not routing traffic to MaaS API

Solution:

  1. Verify HTTPRoute exists and is attached to Gateway: oc get httproute -n redhat-ods-applications

  2. Check HTTPRoute status: oc describe httproute maas-api-route -n redhat-ods-applications

  3. Verify backend service is healthy: oc get svc maas-api -n redhat-ods-applications

  4. Test service directly: oc port-forward svc/maas-api 8080:8080 -n redhat-ods-applications then curl localhost:8080/v1/models
