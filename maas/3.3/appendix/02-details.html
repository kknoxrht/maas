Technical requirements

Software versions

  • Red Hat OpenShift 4.20 or later

  • Red Hat OpenShift AI 3.3 or later

  • OpenShift AI operator channel: stable-3.x

  • NVIDIA GPU operator (latest/stable)

  • OpenShift Service Mesh operator (required for Gateway API)

  • Web browser (Chrome, Firefox, Safari, Edge)

Lab Environemnt Suggestions

Lab Env: A Single Node OpenShift (SNO) cluster with multiple NVIDIA GPUs:

  • Minimum GPU configuration: 1x NVIDIA GPU with 24GB VRAM

  • Recommended GPU configuration: 2x NVIDIA GPUs with 24GB+ VRAM each

  • GPU models supported: A10, A100, L40S, L4, H100 (NVIDIA certified accelerators)

  • Purpose: Deploy multiple models simultaneously (Llama Stack, Nemotron) with distributed inference

Why multiple GPUs matter for this lab:

  • LlamaStack model is required for MaaS Dashboard (consumes 1 GPU)

  • Shows realistic multi-tenant model serving scenarios

  • Enables proper load balancing and autoscaling demonstrations

  • Deploy the NVIDIA-Nemotron-3-Nano-4B-FP8 (or 30B A3B)

Environment setup

Pre-QuickCourse checklist

The following components should be pre-configured before the QuickCourse starts:

□ OpenShift cluster deployed - SNO with GPU nodes configured

□ GPU operators installed - NVIDIA GPU operator and Node Feature Discovery

□ OpenShift AI operator installed - Channel: stable-3.x

□ Service Mesh operator installed - Required for Gateway API routing

□ Red Hat Connectivity Link Operator Installed - Kaudrant - Authentication/Authorization/Rate Limiting

□ Leader Worker Set Operator installed for llm-d - Required for Multiple node Inference (MOE)

□ Web Terminal Operator or Developer Spaces - Required CLI access

□ KServe components deployed - Model serving infrastructure ready

□ GPU driver verification - NVIDIA drivers loaded and tested

□ OpenShift CLI available - oc command accessible from terminal
