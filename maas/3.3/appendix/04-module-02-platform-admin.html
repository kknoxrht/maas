Exercise 1: Enable MaaS in DataScienceCluster

The DataScienceCluster custom resource controls which OpenShift AI components are active. You’ll update this CRD to enable MaaS.

OpenShift AI uses a declarative configuration model. You declare desired state in CRDs, and the operator reconciles actual state to match. This differs from imperative deployment where you manually create pods, services, and configurations.

Understanding the DataScienceCluster CRD

The DataScienceCluster CRD has a components section listing all available features:

  • kserve - Model serving infrastructure (prerequisite for MaaS)

  • modelsAsService - MaaS API, governance, and Gateway integration

  • dashboard - OpenShift AI web console

  • Other components like Pipelines, Workbenches, Model Registry

Each component has a managementState field:

  • Managed - Operator deploys and maintains the component

  • Removed - Operator does not deploy the component

Check current DataScienceCluster state

View the current configuration:

oc get datasciencecluster default-dsc -o yaml

Look for the components section. You’ll likely see:

spec:
  components:
    kserve:
      managementState: Managed
    modelsAsService:
      managementState: Removed
    dashboard:
      managementState: Managed

Note that modelsAsService is currently Removed, meaning MaaS is not deployed.

Enable MaaS component

Edit the DataScienceCluster:

oc edit datasciencecluster default-dsc

Find the modelsAsService section and change managementState from Removed to Managed:

spec:
  components:
    modelsAsService:
      managementState: Managed  # Change this from "Removed" to "Managed"
    kserve:
      managementState: Managed  # Ensure this is also Managed
    dashboard:
      managementState: Managed

Save and exit the editor (:wq in vi, or Ctrl+X then Y in nano).

Monitor MaaS deployment

Watch the operator deploy MaaS components:

oc get pods -n redhat-ods-applications -w | grep maas

You should see pods being created:

maas-api-xxx                  0/1     ContainerCreating   0          5s
maas-default-gateway-xxx      0/1     Init:0/1            0          5s

After 2-5 minutes, all pods should reach Running state:

maas-api-xxx                  1/1     Running   0          3m
maas-default-gateway-xxx      1/1     Running   0          3m

Press Ctrl+C to stop watching.

Verify MaaS API deployment

Check that the MaaS API pod is healthy:

oc get pods -n redhat-ods-applications | grep maas-api

Expected output:

maas-api-7c8f9d5b-xxx         1/1     Running   0          5m

Check the MaaS API service:

oc get svc -n redhat-ods-applications | grep maas

Expected output:

maas-api                ClusterIP   172.30.xxx.xxx   <none>        8080/TCP   5m
maas-default-gateway    ClusterIP   172.30.xxx.xxx   <none>        80/TCP     5m

Verification

Confirm MaaS enablement:

  • DataScienceCluster shows modelsAsService.managementState: Managed

  • MaaS API pod is Running with 1/1 ready containers

  • MaaS service exists in redhat-ods-applications namespace

  • Gateway pod is Running and ready
