Module 4: Governance, Observability, and Chargeback

Estimated time: 5 minutes.

Without tiers and quotas, centralized serving becomes uncontrolled consumption—governance transforms the Centralized Model Hub from free-for-all to fair allocation.

Module Goal

Establish service tiers, enforce request limits, and track token consumption for cross-department billing.

Performance Objective

[PO 3] Implement MaaS Governance and Observability: Define tiered service limits, configure group-based rate limiting, track token consumption metrics, and automate chargeback workflows.

What You’ll Learn

In this module, you will:

  • Create custom governance tiers (Free, Premium, Enterprise)

  • Map OpenShift user groups to tier levels

  • Configure request rate limits and token quotas

  • Calculate chargeback costs from token consumption data

Module Sections

This module contains three sections:

  • 4.1: Defining Governance Tiers - Default tiers and group mappings

  • 4.2: Rate Limits and Quotas - Requests per minute vs token limits

  • 4.3: Lab - Observability & Automated Chargeback - Tracking metrics and billing calculations

What’s Next

With governance and observability in place, the final module shifts perspective to developers—showing how to discover models, generate tokens, and integrate MaaS endpoints into applications.
